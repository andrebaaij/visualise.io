    Template

    Enabling flexible visualisations.

    All visualisations are based on a template, their blueprint, a template consists of multiple elements:

    Metadata
    Template name, description, tags and definition
    Visualisation
    How to draw the visualisation
    Menu
    Make the visiualisation flexible
    Datasets
    Which data are we going to visualise?

    In the next few paragraphs these elements are explained, if you have any questions or feel that this documentation is unclear/incomplete please contact me.

    Metadata

    Name

    Pretty much speaks for itself, this is the name of the template and is a mandatory field.

    Description

    The description of the template is not mandatory, but if you will be publishing your template for other users please make sure the description is clear and consice as to what visualisation can be made with the template.

    Tags

    The tags are used to identify published template in searches, make sure you have the right ones to identify the template.

    Tags are seperated by commas.

    Definition

    In the definition the menu and visualisation are defined. this is done via the _vis javascript API, how this works can be found in the next paragraphs explaining how to draw the visualisation and menu...

    Do not use the javascript _vis object in your definition, as it might work different than expected or break certain functionality.

    Visualisation

    The visualisation definition must be solely defined using the d3.js library and the available parameters.

    draw

    draw(function(_vis.visualisation.DOM, _vis.visualisation.parameters, _vis.datasets)) {
	// Your drawing code goes here
})

    Parameters

    Parameters explained.

    _vis.visualisation.DOM

    The DOM is where you append your svg/visualisation object to.

    draw(function(DOM, parameters, datasets)) {
	// Your drawing code goes here
	svg = DOM.append('svg');
})

    _vis.visualisation.parameters

    This is the object which contians all user editable menu input/parameters (more details are in the next paragraph). The parameter object contains objects for the panel, group and input with a possible table between group and input. For example, a normal input it would be read like this:

    _vis.visualisation.parameters[panelID][groupID][inputID]

    It is also possible that a table contains input(s) these are obtained via the following object:

    _vis.visualisation.parameters[panelID][groupID][tableID][inputID]

    // This examples creates a menu input element in the group dimension
// which is placed in the default panel configuration.
// In the draw function we take the value of this input to define the width of the svg.

group_dimension = panel_configuration.addGroup('dimension')
input_width = group_dimension.addInput('width')
input_width.setLabel('Width')

draw(function(DOM, parameters, datasets)) {
	// Your drawing code goes here
	svg = DOM.append('svg').width(parameters.configuration.dimension.width);
})

    _vis.datasets

    this object ontains all the datasets (only one is supported at the moment) defined for this template. At the moment the default and only dataset is named "Data".

    draw(function(DOM, parameters, datasets)) {
	// Your drawing code goes here
	csvData = datasets.data.data
	data = d3.csv(csvData)
})

    Menu

    Parameterise your visualisation.

    The menu is here to give users the ability to alter their visualisation. Based on normal html inputs such as input, select or textarea the visualisations parameters are filled. The menu is drawn in the order which it is initialised, it consists of the following elements:

    Panel
    Input
    Textarea
    Select
    Input
    Textarea
    Select

    Panel

    A panel is a container which can contain multiple groups. Each panel can also have a label.

    _vis.menu.element.panel(id : String[, heading : String]) //returns _vis.menu.element.panel
_vis.menu.addPanel(id : String[, heading : String]) //returns _vis.menu.element.panel

    // Define our panel and give it a label.
panel_styling = _vis.menu.addPanel("styling","Styling")
    Styling

    The configuration panel

    panel_Configuration is the first panel, if you want to add groups and/or elements to it, feel free to do so.

    Group

    A group can have a label and contain multiple input, select, textarea and table elements, as the name suggests it groups these elements per panel. Multiple groups per Panel are supported and can be handy in some situations.

    _vis.menu.element.group(id : String) //returns _vis.menu.element.group
_vis.menu.element.panel.addGroup(id : String) //returns _vis.menu.element.group

    // This function sets the label for the group
_vis.menu.element.group.setLabel(label : String)

// The next four functions each add a different element 
// to this group, in the next 4 paragraphs these elements are explained more
_vis.menu.element.group.addInput(id : String)
_vis.menu.element.group.addSelect(id : String)
_vis.menu.element.group.addTextarea(id : String)
_vis.menu.element.group.addTable(id : String)

    // First define the panel which we are going to add the groups to:
panel_styling = _vis.menu.addPanel("styling","Styling")	

// Let's add our first group, which will contain inputs for styling text:
group_text = panel_styling.addGroup("text")
group_text.setLabel("Text")
input_textColor = group_text.addInput("color")
input_textColor.setLabel("Color")
// And our second and third which are groups for styling borders and the visualisation itself.
group_border = panel_styling.addGroup("border")
group_border.setLabel("Border")
input_borderColor = group_border.addInput("color")
input_borderColor.setLabel("Color")

group_visualisation = panel_styling.addGroup("visualisation")
group_visualisation.setLabel("Visualisation")
input_visualisationColor = group_visualisation.addInput("color")
input_visualisationColor.setLabel("Color")
    Styling
    Color
    Color

    Table

    Tables can be used to group sets of inputs together, a table always consists of two columns. The first column can only contain text, the second column can contain text or an input/select/textarea.

    _vis.menu.element.table(id : String) //returns _vis.menu.element.table
_vis.menu.element.group.addTable(id : String) //returns _vis.menu.element.table

    // Set the headers for the table
_vis.menu.element.table.setHeaders(header1 : String, header2 : String) 

// Add a row to the table
_vis.menu.element.table.addRow(col1 : String, col2 : {String,Input,Select,TextArea})

    // Add the panel
panel_styling = _vis.menu.addPanel("styling","Styling")
// And the group
    group_fill = panel_styling.addGroup("fill")
    group_fill.setLabel("Fill:")

// Add the table with inputs:
      table_background = group_fill.addTable("background")
        // Set the headers
        table_background.setHeaders('Age','Color')
        // Add the different inputs
        input1 = _vis.menu.element.input("agegroup1")
        table_background.addRow('0-20', input1)

        input2 = _vis.menu.element.input("agegroup2")
        table_background.addRow('21-50', input2)

        input3 = _vis.menu.element.input("agegroup3")
        table_background.addRow('51-99', input3)

    Input

    This menu element is the standard html input, with an added label.

    _vis.menu.element.input(id : String) //returns _vis.menu.element.input
_vis.menu.element.group.addInput(id : String) //returns _vis.menu.element.input

    _vis.menu.element.input.setLabel(label : String)

    // First define the panel which we are going to add the group/input to
panel_styling = _vis.menu.addPanel("styling","Styling")	

// Let's add a group
group_text = panel_styling.addGroup("text")

// And here is our input:
input_textColor = group_text.addInput("color")
input_textColor.setLabel("Color")
    Styling
    Color

    Select

    This menu element is the standard html select, with an optional label. Every select must have one or more options in order to be usefull, these can be added via this function. Provide a value and a label which the user can select.

    _vis.menu.element.select(id : String) //returns _vis.menu.element.select
_vis.menu.element.group.addSelect(id : String) //returns _vis.menu.element.select

    _vis.menu.element.select.setLabel(label : String)
_vis.menu.element.select.addOption(value : String, label : String)

    // First define the panel which we are going to add the group/input to
panel_styling = _vis.menu.addPanel('styling','Styling')	

// Let's add a group
group_text = panel_styling.addGroup('text')

// And here is our input:
select_textColor = group_text.addSelect('color')
select_textColor.setLabel('Color')
select_textColor.addOption('#fff','White')
select_textColor.addOption('#ccc','Grey')
select_textColor.addOption('#000','Black')
    Styling
    Color

    _vis.menu.element.textarea.setLabel(label : String)

    // First define the panel which we are going to add the group/input to

// Let's add a group

// And here is our input:

    DataSets

    What will be visualised?

    Most of the visualisations will want to visualise some data, any data. At the moment only one default dataset named "data" is supported.

    Data

    In the data tab it is possible to add comma seperated data, after that it is available as a parameter in the draw function for use. Some basic information on the datasets are available through functions or arrays. If you have any suggestions for other functions data, please let us know!

    draw(function(DOM, parameters, datasets)) {
	// Your drawing code goes here
	csvData = datasets.data.data
	data = d3.csv(csvData)
})
